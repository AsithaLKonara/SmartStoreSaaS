## Severity Guidelines

### Critical
- Data loss or corruption
- Security breach
- System crash
- Complete feature failure
- Data leakage between tenants
- Payment processing failure

### High
- Major feature broken (workaround exists)
- Incorrect calculations (financial impact)
- Performance issues (major degradation)
- Authentication/authorization failures

### Medium
- Minor feature issue (easy workaround)
- UI/UX problems
- Non-critical validation issues
- Minor performance issues

### Low
- Cosmetic issues
- Typos
- Minor UI inconsistencies
- Non-functional improvements

---

## Status Guidelines

- **Open**: Issue found, not yet addressed
- **In Progress**: Currently being fixed
- **Fixed**: Code fix completed, needs verification
- **Verified**: Fix verified by QA, issue closed
- **Deferred**: Issue acknowledged but deferred to future release
